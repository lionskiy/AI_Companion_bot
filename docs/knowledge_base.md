# Mirror — База знаний (Knowledge Base)

## Структура коллекций Qdrant

### Системные коллекции (память пользователей)

| Коллекция | Назначение |
|-----------|-----------|
| `user_episodes` | L2 память: резюме сессий диалогов |
| `user_facts` | L3 память: извлечённые факты о пользователях |

Фильтрация по `user_id` в payload — все пользователи в одной коллекции.

## Формат векторов

- Модель эмбеддингов: `text-embedding-3-large` (OpenAI)
- Размерность: **3072**
- Метрика: **Cosine**
- Каждый чанк хранится в двух версиях: оригинал + перевод

### Payload полей

```json
{
  "topic": "КПТ тревога / Книга Лихи часть 1",
  "text": "текст чанка",
  "collection": "knowledge_psych",
  "lang": "ru"
}
```

Поле `lang` принимает `"ru"` или `"en"`.

---

## Билингвальный ингест

При загрузке материала система автоматически:

1. Определяет язык по доле кириллицы в первых 500 символах (>20% = RU, иначе EN)
2. Переводит все чанки на второй язык через LLM (`intent_classify` task_kind)
3. Эмбеддит оба варианта параллельно
4. Записывает оба в Qdrant с полем `lang`

Параметр `source_lang` в запросах инgeста:
- `"auto"` — автоопределение (по умолчанию)
- `"ru"` — принудительно считать исходный русским
- `"en"` — принудительно считать исходным английским

## Загрузка материалов

### Через Admin UI

**Вкладка URL** — вставить ссылку, указать коллекцию и язык.

**Вкладка Файл** — загрузить файл напрямую. Поддерживаемые форматы:
- `.epub`, `.fb2` — электронные книги
- `.pdf` — PDF-документы  
- `.docx` — Word-документы
- `.txt`, `.md` — текст
- `.zip` — архив с несколькими файлами

**Вкладка Датасет** — загрузить датасет по URL (GitHub raw, HuggingFace, прямые ссылки).  
Форматы: JSON, JSONL, CSV. Автоопределение полей вопрос/ответ.

### ZIP с автороутингом по коллекциям

Организовать книги по папкам, названным как Qdrant коллекции:

```
books.zip
├── knowledge_psych_cbt/
│   ├── burns_feeling_good.epub
│   └── leahy_anxiety_free.epub
├── knowledge_psych_trauma/
│   └── pete_walker_cptsd.epub
└── knowledge_stories/
    └── bakman_anxious_people.epub
```

При загрузке такого ZIP:
- Система читает имя верхней папки
- Если начинается с `knowledge_` — роутит файл в эту коллекцию
- Коллекция создаётся автоматически если не существует
- Файлы без папки или в папке без `knowledge_` → в коллекцию из формы

## KB Routing (планируется в Stage 2)

Таблица `kb_routing` в PostgreSQL: `intent → List[collection_name]`

Пример:
```
cbt              → [knowledge_psych_cbt]
psychoanalysis   → [knowledge_psych_deep]
trauma           → [knowledge_psych_trauma]
relationships    → [knowledge_psych_relationships]
general_psych    → [knowledge_psych_cbt, knowledge_psych_selfhelp, knowledge_psych_articles]
tarot            → [knowledge_tarot]
astrology        → [knowledge_astro]
narrative        → [knowledge_stories]
```

Настраивается из Admin Panel без деплоя. `intent_classify` определяет topic, диалог-сервис делает RAG только в релевантные коллекции.
