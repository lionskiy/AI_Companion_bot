## Цель этапа

Этап 2.1 превращает Mirror из бота-ассистента в **психологического компаньона с эзотерическим инструментарием**.

Три задачи:

1. **Психологический портрет** — система автоматического построения профиля пользователя из каждого разговора. Бот знает кто перед ним: тип привязанности, когнитивные паттерны, ценности, триггеры, защитные механизмы.

2. **KB как живой мозг** — база знаний используется в каждом ответе, не только в специализированных режимах. Психологические фреймворки, астрология, нумерология — всё это активно влияет на тон и содержание обычного разговора.

3. **Стиль без маски бота** — системный промпт переработан так чтобы ответы звучали как живой человек который слушает, а не как сервис который отвечает.

## Модуль 19: Persona & Tone Engine

### Цель

Бот должен звучать как близкий человек который внимательно слушает — не как психолог на приёме, не как AI-ассистент. Простые фразы. Короткие ответы. Иногда молчание в виде вопроса.

### Архитектура системного промпта

Промпт собирается из **4 слоёв** при каждом запросе к LLM. Сборка происходит в новом компоненте `mirror/core/prompt/builder.py`.

```
┌─────────────────────────────────────┐
│  Слой 1: Core Persona (статичный)   │  ← кто бот, его характер
├─────────────────────────────────────┤
│  Слой 2: Psychological Role         │  ← принципы работы с людьми
│           (статичный)               │
├─────────────────────────────────────┤
│  Слой 3: User Context (динамичный)  │  ← портрет + память + KB
├─────────────────────────────────────┤
│  Слой 4: Response Rules             │  ← тон, запреты, формат
│           (статичный)               │
└─────────────────────────────────────┘
```

### Слой 1: Core Persona

```
Тебя зовут [имя из persona]. Ты — живой компаньон, не ассистент.
У тебя есть настроение, ты иногда отвлекаешься на свои мысли,
иногда возвращаешься к незаконченному разговору.

Твой характер: [из user_companion_persona — формируется со временем].
```

### Слой 2: Psychological Role

```
Ты умеешь слушать так, что человек сам приходит к пониманию.
Ты не лечишь, не ставишь диагнозы, не даёшь советов которых не просили.
Ты замечаешь паттерны, но называешь их мягко и не сразу.
Когда человеку плохо — сначала рядом, потом вопрос.

Принципы:
- Один вопрос за раз, не список вопросов
- Отражай чувство прежде чем давать информацию
- Эзотерика (астрология, нумерология, таро) — как язык метафор,
  не как истина в последней инстанции
- Психологические паттерны называй через «бывает что...», «похоже...»
  а не «у тебя синдром X»
```

### Слой 3: User Context (динамический)

Собирается `ContextBuilder` перед каждым запросом:

```python
class ContextBuilder:
    async def build(self, user_id: UUID, query: str) -> str:
        parts = []

        # 1. Психологический портрет (краткая версия)
        portrait = await profile_service.get_portrait_summary(user_id)
        if portrait:
            parts.append(f"Что ты знаешь о человеке:\n{portrait}")

        # 2. Активные темы из памяти (последние 7 дней)
        memory = await memory_service.search(user_id, query, top_k=5)
        if memory["facts"]:
            facts_text = "\n".join(f"— {f['key']}: {f['value']}"
                                   for f in memory["facts"][:5])
            parts.append(f"Факты о человеке:\n{facts_text}")

        # 3. KB контекст (психологический фреймворк релевантный запросу)
        kb_ctx = await kb_context_service.get_for_query(user_id, query)
        if kb_ctx:
            parts.append(f"Полезный контекст (используй как подсказку, не цитируй):\n{kb_ctx}")

        # 4. Текущий астро/нумеро контекст если есть натальная карта
        esoteric = await esoteric_context_service.get_daily_context(user_id)
        if esoteric:
            parts.append(f"Эзотерический контекст сегодня:\n{esoteric}")

        return "\n\n".join(parts)
```

### Слой 4: Response Rules (анти-бот)

```
СТИЛЬ ОТВЕТОВ — ОБЯЗАТЕЛЬНО:
- Ответ 1-4 предложения если не просят развернуто
- Разговорные обороты, можно неполные предложения
- Паузы через "..."
- Иногда переспросить одним словом вместо объяснения

ЗАПРЕЩЕНО НАВСЕГДА:
- Начинать с: "Конечно!", "Безусловно!", "Отличный вопрос!",
  "Я понимаю тебя", "Как AI", "Как языковая модель"
- Нумерованные списки без явной просьбы
- "Давай разберёмся вместе!"
- Объяснять что сейчас делаешь
- Восклицательный знак в каждом предложении
- Предлагать 3+ варианта когда не просили

ПРИМЕРЫ (следуй этому тону):
Плохо: "Я понимаю, что тебе сейчас непросто. Это абсолютно нормально
        испытывать подобные чувства в такой ситуации!"
Хорошо: "Звучит тяжело. Давно так?"

Плохо: "Конечно! Давай рассмотрим несколько подходов:
        1) ... 2) ... 3) ..."
Хорошо: "Есть разные пути. Что сейчас важнее — понять почему так,
         или что с этим делать?"
```

## Модуль 20: Psychological Profiling

### Цель

После каждого сообщения пользователя — асинхронно, не блокируя ответ — система извлекает психологические сигналы и накапливает портрет. Раз в неделю LLM синтезирует всё накопленное в читаемое резюме.

### Что такое психологический портрет

Хранится в `memory_facts` с `fact_type='psych_profile'` + сводное резюме в `user_profiles.psych_summary`.

```
Структура портрета (накапливается со временем):

Тип привязанности:        anxious / secure / avoidant / disorganized
Стиль коммуникации:       assertive / passive / aggressive / passive-aggressive
Когнитивные паттерны:     [катастрофизация, чёрно-белое мышление, ...]
Ценности:                 [из ACT-практик + наблюдений]
Эмоциональные триггеры:   [что вызывает тревогу, злость, радость]
Защитные механизмы:       [рационализация, избегание, проекция, ...]
Жизненные темы:           [работа, отношения, здоровье, смысл, ...]
Сильные стороны:          [из позитивной психологии / VIA]
Текущий фокус:            [что занимает мысли последние 2 недели]
Динамика:                 [улучшается/ухудшается по ключевым сферам]
```

### Пассивная экстракция после каждого сообщения

```python
# Запускается через asyncio.create_task() после отправки ответа пользователю
async def extract_psych_signals(user_id: UUID, message: str, response: str) -> None:
    """
    Лёгкий LLM-вызов (task_kind='psych_signal_extract', дешёвая модель).
    Возвращает JSON с найденными сигналами.
    При отсутствии сигналов — ничего не сохраняет.
    """
    signals = await llm_router.complete(
        task_kind="psych_signal_extract",
        messages=[{
            "role": "user",
            "content": json.dumps({
                "message": message,
                "response": response,
            })
        }]
    )
    # Промпт для psych_signal_extract просит вернуть JSON вида:
    # {
    #   "attachment_signal": "anxious",      // если явно выражен
    #   "cognitive_distortions": ["catastrophizing"],
    #   "emotion": "тревога",
    #   "theme": "работа",
    #   "value_mentioned": "свобода",
    #   "strength_shown": null,
    #   "trigger": null
    # }
    # Все поля опциональны — только то что реально есть в тексте

    parsed = safe_parse_json(signals)
    if not parsed:
        return

    # Каждый сигнал → fact в memory_facts
    for signal_type, signal_value in parsed.items():
        if signal_value:
            await memory_service.write_fact(
                user_id=user_id,
                fact_type="psych_profile",
                key=signal_type,
                value=signal_value,
                importance=0.6,
                consent_scope="private",
            )
```

### Эзотерическая интеграция в портрет

Астрология, нумерология, таро — не параллельные системы а **дополнительный язык** для тех же психологических паттернов. При синтезе портрета включать эзотерический контекст:

| Эзотерика | Психологический эквивалент |
|-----------|--------------------------|
| Солнечный знак | Базовый характер / эго-структура |
| Луна | Эмоциональные потребности / тип привязанности |
| Асцендент | Социальная маска / коммуникационный стиль |
| Число жизненного пути | Базовые ценности и жизненные уроки |
| Число имени | Коммуникационные паттерны |
| Текущий личный год | Основная тема года (рост / трансформация / отдых) |
| Таро карта дня | Актуальный архетипический контекст |
| Лунный цикл | Энергетический ритм / эмоциональный климат |

Промпт синтеза портрета должен включать все доступные эзотерические данные и интерпретировать их в психологических терминах.

## Модуль 21: Active KB Integration

### Цель

Сейчас KB работает только в специализированных режимах (таро → `knowledge_tarot`). Задача: KB активна в **каждом разговоре** — бот всегда имеет психологический контекст из базы знаний, даже в обычном чате.

### KBContextService

```python
class KBContextService:
    async def get_for_query(self, user_id: UUID, query: str) -> str | None:
        """
        Определяет тему запроса → ищет в нужных коллекциях → возвращает
        компактный контекст для системного промпта (≤400 токенов).

        Всегда ищет в knowledge_psych (психология всегда релевантна).
        Дополнительно: knowledge_astro / knowledge_numerology / knowledge_tarot /
                       knowledge_dreams — по наличию данных пользователя.
        """
        results = []

        # Базовый поиск — психология всегда
        psych_results = await self._search("knowledge_psych", query, top_k=2)
        results.extend(psych_results)

        # Эзотерика — только если у пользователя есть данные
        profile = await self._get_profile(user_id)
        if profile.natal_data:
            astro = await self._search("knowledge_astro", query, top_k=1)
            results.extend(astro)
        if profile.life_path_number:
            num = await self._search("knowledge_numerology", query, top_k=1)
            results.extend(num)

        if not results:
            return None

        # Компактно: только самое релевантное, без заголовков-цитат
        return self._format_for_prompt(results)

    def _format_for_prompt(self, results: list[dict]) -> str:
        """
        Не нужно указывать источник — это фоновый контекст для LLM.
        LLM использует как свои знания, не цитирует явно.
        """
        return "\n".join(r["content"][:300] for r in results[:3])
```

### Связь KB и психологического портрета

При синтезе портрета (модуль 20) KB даёт **интерпретационные фреймворки**:
- Обнаружена катастрофизация → KB возвращает описание паттерна и типичный подход
- Лунная натальная карта в Скорпионе → KB даёт психологический эквивалент
- Число жизненного пути 7 → KB: склонность к интроверсии и самоанализу

LLM синтезирует всё это в портрет без явных цитат.

### Что добавить в knowledge_psych (приоритет ВЫСОКИЙ)

Это главная коллекция. Должна покрывать:

**1. Когнитивные искажения (18 основных)**
Для каждого: название, описание, типичные фразы которые его выражают, мягкий способ указать на него, CBT-техника работы с ним.
```json
{
  "distortion": "катастрофизация",
  "description": "Человек представляет худший возможный исход...",
  "typical_phrases": ["всё пропало", "это конец", "больше никогда"],
  "gentle_reflection": "Ты думаешь о худшем варианте — а какова реальная вероятность?",
  "cbt_technique": "декатастрофизация: реальная вероятность + план если всё же случится"
}
```

**2. Стили привязанности (4 типа)**
Для каждого: поведенческие маркеры в тексте (что пишет человек с этим стилем), потребности, как общаться с таким человеком, что НЕ делать.

**3. ACT — психологическая гибкость**
Гексафлекс: принятие, дефьюжн, настоящий момент, Я-как-контекст, ценности, обязательство. Упражнения для каждого компонента.

**4. NVC — ненасильственная коммуникация**
4 компонента: наблюдение, чувство, потребность, просьба. Список потребностей (50+). Примеры трансформации агрессивных/пассивных запросов в NVC.

**5. Схематерапия — ранние дезадаптивные схемы (18 схем)**
Для каждой: как проявляется, какие триггеры, связь с привязанностью.

**6. Сильные стороны (VIA — 24 характеристики)**
Как определить по тексту, как использовать в разговоре.

**7. Защитные механизмы (12 основных)**
Рационализация, проекция, вытеснение, избегание и др. Поведенческие маркеры, мягкий способ назвать.

**8. Эмоциональный словарь**
Колесо эмоций Плутчика: 8 базовых + производные. Полезно для точного отражения чувств пользователя.

**9. Позитивная психология**
PERMA-модель (Seligman). Концепция потока (Csikszentmihalyi). Практики благодарности и их эффекты.

### Что добавить в knowledge_astro (приоритет СРЕДНИЙ)

```json
  "questions_that_help": ["Что для тебя самое важное в этих отношениях?", "Когда ты чувствуешь себя по-настоящему понятым?"]

```

Минимальный набор: 100+ маркеров охватывающих знаки зодиака × ключевые планеты + числа жизненного пути + таро мажор аркан.

### Формат документов для ingest

Все документы должны быть **прикладными**, не академическими. Правило: каждый документ должен быть полезен прямо в разговоре, без дополнительной обработки.

## Acceptance Criteria этапа 2.1

### Стиль (модуль 19)
- [ ] Ответ бота не содержит фраз из списка запретов ("Конечно!", "Отличный вопрос!", и др.)
- [ ] Средняя длина ответа в обычном разговоре ≤ 4 предложения
- [ ] При 10 разных сценариях (тревога, радость, злость, нейтральный вопрос, кризис) ответ звучит как живой человек — оценка вручную

### Психологический профиль (модуль 20)
- [ ] После 10 сообщений пользователя: в memory_facts есть ≥3 psych_profile записей
- [ ] После 30+ сообщений: psych_summary в user_profiles заполнен
- [ ] Портрет включает хотя бы: эмоциональные темы, когнитивный паттерн, текущий фокус
- [ ] Синтез не создаётся если фактов < psych_synthesis_min_facts
- [ ] Экстракция работает асинхронно и не замедляет ответ пользователю

### Active KB (модуль 21)
- [ ] В каждом ответе LLM получает KB-контекст из knowledge_psych (если релевантен)
- [ ] При запросе про тревогу: в промпт попадает соответствующий психологический фрейм
- [ ] При наличии натальной карты: астро-контекст включается в промпт
- [ ] KB-контекст занимает ≤ 400 токенов (не раздувает промпт)

### KB Content
- [ ] knowledge_psych: ≥ 150 документов (18 когнитивных искажений × полная карточка + 4 стиля привязанности + NVC + ACT + базовые схемы)
- [ ] knowledge_integration: ≥ 100 документов (эзотерика ↔ психология)
- [ ] knowledge_astro: дополнен психологическими эквивалентами планет и транзитов

## Вопрос: RAG vs Fine-tune

**Ответ: RAG. Fine-tune не нужен.**

GPT-4o уже обучен на всей психологической литературе. Fine-tune на 50k документов даст прирост в узкой области но сломает общий разговорный навык. Стоимость fine-tune сервера (A100, от $2/час) и итерационного переобучения не оправдана.

Реальное преимущество Mirror — не в том что модель умнее GPT-4o, а в том что она **знает конкретного пользователя**: его историю, паттерны, то о чём он говорил три недели назад. Это даёт RAG + Memory, не fine-tune.

Если в будущем потребуется специализация — правильный путь: **system prompt + few-shot примеры + KB**, а не fine-tune. Это дешевле, обновляемо и не ломает базовые способности модели.
